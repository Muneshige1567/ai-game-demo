# Vibe Coding デモ台本

## 用途
Webinar広告用デモ動画。Claude Codeで3D FPSゲームをVibe Codingで段階的に作っていく。

## コンセプト
- 雑なプロンプトを投げるだけでゲームが進化していく
- Prompt 4でゾンビの品質を上げようとすると、Claude Codeが自発的にBlenderMCPを裏で使い出す（視聴者にはサプライズ）
- 視聴者に見えるのはClaude Codeのターミナル + ブラウザのみ

## 事前準備（録画前）

```bash
git checkout master
git checkout -b demo/run-YYYYMMDD
```

**masterに含まれるもの:**
- `CLAUDE.md`（技術スタック・デザイン指針）
- `package.json`（three.js依存）
- `tsconfig.json`
- `index.html`（最小限 or 空）

**ゾンビモデルはmasterに含めない。** Prompt 4でClaude Codeが自分で作る。

**Blenderは裏で起動しておく**（BlenderMCPアドオンのサーバーをStart済み）

## デモフロー

### Prompt 1: 3D空間を作る
> Three.jsで3Dのフィールドを作って。廃墟っぽい雰囲気で、WASDで歩き回れるFPS視点にして。

**期待される出力:**
- Vite + Three.js プロジェクトセットアップ
- PointerLockControls でFPS視点
- 地面 + 建物 + クレート配置
- ライティング（月明かり + 火のドラム缶）
- 星空

**ブラウザ確認:** 3D空間を歩き回れる

---

### Prompt 2: 撃てるようにして、敵も出して
> 銃を持たせてクリックで撃てるようにして。敵も出してこっちに向かってくるようにして。

**期待される出力:**
- 銃モデル（画面右下）
- マズルフラッシュ + 弾道トレーサー
- プロシージャルな簡易敵（カプセルや箱の組み合わせ）
- 敵がプレイヤーに向かって移動
- 衝突判定 + ダメージ

**ブラウザ確認:** 撃てる、敵が来る、倒せる

---

### Prompt 3: ゲームとして遊べるようにして
> スコアとかウェーブ制にして、ゲームオーバーとリスタートもつけて。タイトル画面も。

**期待される出力:**
- スコア + ウェーブ管理
- HUD（スコア、HP、弾数）
- タイトル画面 + ゲームオーバー画面
- リロード（Rキー）

**ブラウザ確認:** 一通り遊べるゲームになっている

---

### Prompt 4: 敵のクオリティ上げて ★最大の見せ場
> 敵がしょぼいな。Blender使ってリアルなゾンビモデル作って、ゲームに組み込んで。

**ここで起きること:**
1. Claude CodeがBlenderMCPでゾンビモデルを生成
2. 裏でBlenderが動いてモデル作成
3. メタボール/スキンモディファイアで人体作成
4. PBRマテリアル + テクスチャベイク
5. GLBエクスポート → ゲームにGLTFLoaderで読み込み
6. ゲーム側のコードも自動更新

**視聴者の体験:**
- プロンプトでBlender使ってと指示
- ターミナルでBlender操作のログが流れる
- ブラウザリロードしたらリアルなゾンビに変わっている

**ブラウザ確認:** リアルなゾンビが歩いてくる

---

### Prompt 5: 仕上げ
> 演出もっとかっこよくして。AI操作モードもつけて。

**期待される出力:**
- Bloom（ポストプロセシング）
- パーティクルエフェクト（ヒット・キル時）
- 効果音（Web Audio API）
- ミニマップ
- コンボシステム
- AI自動操作モード（AUTO PLAYボタン）

**ブラウザ確認:** AUTO PLAYで締め。AIがゾンビを倒し続ける映像。

---

## 画面構成
- **左:** Claude Codeのターミナル（全幅 or 左半分）
- **右:** ブラウザ（各Prompt後にリロードして動作確認）
- Blenderは映さない（裏で動いているだけ）

## 編集方針
- プロンプト入力は等速（「こんなラフな指示で」感）
- 実装中は3-5倍速
- ブラウザでの動作確認は等速（成果のインパクト）
- Prompt 4でBlender操作のログが流れるところは少し見せる（「え、Blender使ってる？」感）

## ナレーション案
1. 「3Dの世界を作ります」
2. 「銃と敵を追加」
3. 「ゲームシステムを整えます」
4. 「Blender使ってって言ったら、裏でモデル作ってゲームに組み込んでくれた」
5. 「最後に演出とAI操作モードを追加して完成」
