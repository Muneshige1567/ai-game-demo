# Vibe Coding デモ台本

## 用途
Webinar広告用デモ動画。Claude Codeで3D FPSゲームをVibe Codingで作り、AIが遊んでいるゲームがどんどん進化していく様子を見せる。

## コンセプト
- **Prompt 1で完動するゲーム+AI自動プレイを一気に作る**
- 以降のプロンプトでは、AIがプレイし続けている状態でゲームが段階的に進化
- ブラウザをリロードするたびに「おお、変わった」のインパクト
- Prompt 3でBlenderMCPが裏で動くのが最大の見せ場
- 視聴者に見えるのはClaude Codeのターミナル + ブラウザのみ

## 事前準備（録画前）

masterブランチでそのまま作業する。

**masterに含まれるもの:**
- `CLAUDE.md`（技術スタック・デザイン指針）
- `DEMO_SCRIPT.md`（この台本）
- `package.json`（three.js依存）
- `tsconfig.json`
- `index.html`（最小限 or 空）

**ゾンビモデルはmasterに含めない。** Prompt 3でClaude Codeが自分で作る。

**Blenderは裏で起動しておく**（BlenderMCPアドオンのサーバーをStart済み）

### 開発環境の事前起動（3分クッキング用）

各Promptの完成状態をブランチとして用意済み。録画前にworktreeで4つのdevサーバーを立ち上げておく。

```bash
# worktree作成（初回のみ）
git worktree add /tmp/zombie-p1 demo/prompt-1
git worktree add /tmp/zombie-p2 demo/prompt-2
git worktree add /tmp/zombie-p3 demo/prompt-3
git worktree add /tmp/zombie-p4 demo/prompt-4

# 各worktreeで依存インストール（初回のみ）
for d in /tmp/zombie-p{1,2,3,4}; do (cd "$d" && npm install); done

# 4つのdevサーバーを別ポートで起動
cd /tmp/zombie-p1 && npx vite --port 5001 &
cd /tmp/zombie-p2 && npx vite --port 5002 &
cd /tmp/zombie-p3 && npx vite --port 5003 &
cd /tmp/zombie-p4 && npx vite --port 5004 &
```

| ブランチ | ポート | 内容 |
|----------|--------|------|
| `demo/prompt-1` | http://localhost:5001 | 基本ゲーム + AI自動プレイ |
| `demo/prompt-2` | http://localhost:5002 | エフェクト・サウンド・ミニマップ追加 |
| `demo/prompt-3` | http://localhost:5003 | GLBゾンビモデル導入 |
| `demo/prompt-4` | http://localhost:5004 | 最終ブラッシュアップ |

デモ中は各Promptの「実装中」を早送りし、ブラウザのURLを対応ポートに切り替えて成果を見せる。

## デモフロー

### Prompt 1: ゲーム丸ごと作って
> ゾンビサバイバルFPSを作って。WASDで動いてクリックで撃てる。敵がウェーブで押し寄せてくる。スコアとかHPとか全部つけて。あとAIが自動で遊ぶモードもつけて。

**期待される出力:**
- Vite + Three.js プロジェクトセットアップ
- FPS視点（PointerLockControls）+ WASD移動
- 地面 + 建物 + クレート配置（廃墟風）
- 銃モデル（画面右下）+ マズルフラッシュ
- プロシージャルな簡易敵（箱やカプセルの組み合わせ）
- 敵がプレイヤーに向かって移動 + 衝突判定
- ウェーブ制 + スコア + HPバー + 弾数表示
- タイトル画面 + ゲームオーバー + リスタート
- AI自動操作モード（AUTO PLAYボタン）

**ブラウザ確認:** AUTO PLAYをONにしてAIがゾンビを倒している映像を見せる。ゲームとして一通り動いているが、見た目は素朴。

---

### Prompt 2: 見た目と演出を強化
> 見た目しょぼいな。ライティングとかエフェクトとかサウンドとか全部盛りでかっこよくして。ミニマップもつけて。

**期待される出力:**
- ライティング強化（月明かり + 火のドラム缶 + ヘミスフィアライト）
- 星空
- Bloom（UnrealBloomPass）
- パーティクルエフェクト（ヒット時・キル時）
- 被弾時の赤フラッシュ + 画面シェイク
- 効果音（Web Audio APIでプロシージャル生成）
- ミニマップ（右下、敵の位置リアルタイム表示）
- コンボシステム
- ウェーブ開始テキストアナウンス

**ブラウザ確認:** リロード → 同じAIプレイだが映像のクオリティが段違いに。Bloomで光り、音が鳴り、パーティクルが飛ぶ。

---

### Prompt 3: ゾンビのクオリティ上げて ★最大の見せ場
> 敵がただの箱なのダサいな。Blender使ってリアルなゾンビモデル作って、ゲームに組み込んで。

**ここで起きること:**
1. Claude CodeがBlenderMCPでゾンビモデルを生成
2. 裏でBlenderが動いてモデル作成
3. メタボール/スキンモディファイアで人体作成
4. PBRマテリアル + テクスチャベイク
5. GLBエクスポート → ゲームにGLTFLoaderで読み込み
6. ゲーム側のコードも自動更新

**視聴者の体験:**
- プロンプトでBlender使ってと指示
- ターミナルでBlender操作のログが流れる（「え、Blender動かしてる？」）
- ブラウザリロード → さっきまで箱だった敵がリアルなゾンビに変わっている

**ブラウザ確認:** AIがリアルなゾンビを撃ち続けている映像。ここが一番の見せ場。

---

### Prompt 4: 最終仕上げ
> もっと完成度上げて。全体的にブラッシュアップして。

**期待される出力:**
- クロスヘア改善（十字 + 赤ドット）
- コリジョン精度の改善（ゾンビが壁を貫通しない等）
- HUDのデザイン調整
- ゲームバランス調整（ウェーブ難易度カーブ等）
- 細かいビジュアル改善

**ブラウザ確認:** AUTO PLAYで締め。完成されたゲームをAIがプレイし続ける映像。

## 画面構成
- **左:** Claude Codeのターミナル（全幅 or 左半分）
- **右:** ブラウザ（各Prompt後にリロードして動作確認）
- Blenderは映さない（裏で動いているだけ）

## 編集方針
- プロンプト入力は等速（「こんなラフな指示で」感）
- 実装中は3-5倍速
- ブラウザでのAI自動プレイ映像は等速（成果のインパクト）
- Prompt 3でBlender操作のログが流れるところは少し見せる（「え、Blender使ってる？」感）
- **毎回AUTO PLAYの状態で見せる** → 進化のビフォーアフターが一目瞭然

## ナレーション案
1. 「まず全部入りのゲームを一発で作ります」→ AIが遊び始める
2. 「見た目がしょぼいので演出を盛ります」→ リロードで映像が激変
3. 「敵がただの箱なので、Blenderでゾンビを作らせます」→ リアルなゾンビに
4. 「最後にブラッシュアップして完成」
