# Zombie Survival - 3D FPS Game

## 技術スタック
- Vite + TypeScript
- Three.js（3Dレンダリング）
- Web Audio API（効果音）
- dev server: `npm run dev`

## プロジェクト構成
```
public/
  zombiea.glb    # ゾンビモデルA（BlenderMCPで作成済み）
  zombieb.glb    # ゾンビモデルB
  zombiec.glb    # ゾンビモデルC
src/
  main.ts        # ゲーム全体（単一ファイル）
index.html       # HUD・UI
```

## ゲーム概要
- FPS視点のゾンビサバイバルシューター
- WASD移動、マウス視点操作、左クリック射撃、Rリロード
- ウェーブ制で敵が増加
- AI自動操作モード搭載

## デザイン指針

### 明るさ・視認性（最重要）
- 画面が暗すぎると何も見えない。トーンマッピング露出は1.4以上
- アンビエントライト強め（1.0以上）、ヘミスフィアライト併用
- 霧は薄く（FogExp2 density 0.014以下）
- ゾンビは暗い背景に対して視認できる色にする

### ゾンビモデル
- `public/` に配置済みのGLBファイルを `GLTFLoader` で読み込む
- スポーン時にランダムで3種類から選択
- コライダー内にスポーンしないようチェック
- 敵が近づくまでAIは撃たない（距離12以内で射撃開始）

### エフェクト・演出
- Bloom（UnrealBloomPass）控えめに（0.35程度）
- マズルフラッシュ + PointLight で発射時の光
- パーティクルはヒット時・キル時に出す
- 被弾時の赤フラッシュ + 画面シェイク
- ウェーブ開始時にテキストアナウンス

### HUD
- 左上: スコア + ウェーブ
- 右上: 弾数
- 左下: HPバー
- 右下: ミニマップ（敵の位置をリアルタイム表示）
- 中央: クロスヘア（十字 + 赤ドット）

### コリジョン
- 全ての障害物（建物・クレート）に衝突判定
- プレイヤー・ゾンビ両方が障害物を貫通しない
- ゾンビはプレイヤーに近づくと停止、密着しすぎたら押し戻す

### サウンド
- Web Audio APIでプロシージャル生成（外部ファイル不使用）
- AudioContextはクリック時にresumeする（ブラウザポリシー対策）

## やってはいけないこと
- ファイル分割しない（src/main.ts に全て書く）
- 外部アセットのダウンロードURL指定しない
- 画面を暗くしすぎない
- ゾンビの射撃距離を遠くしすぎない（AIデモ時にゾンビが見える前に倒される）
