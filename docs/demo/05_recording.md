# 05. 画面録画ガイド

## 録画ソフト

### OBS Studio（推奨・無料）
- https://obsproject.com/
- Windows/Mac 対応
- シーン切り替え、ウィンドウキャプチャが柔軟

### 代替
- Windows: Xbox Game Bar（Win + G）— シンプルだが画面構成が制限される
- Mac: QuickTime / ScreenFlow

---

## OBS 設定

### 1. 出力設定
- 設定 → 出力
  - 出力モード: 詳細
  - エンコーダ: x264 または NVENC（GPU搭載の場合）
  - ビットレート: 6000 Kbps（YouTube想定）
  - キーフレーム間隔: 2
- 設定 → 映像
  - 基本解像度: 1920x1080
  - 出力解像度: 1920x1080
  - FPS: 30

### 2. シーン構成

3つのソースを配置する:

```
┌──────────────────────────────────────────┐
│                                          │
│  ┌─────────────┐  ┌─────────────────┐   │
│  │             │  │                 │   │
│  │ Claude Code │  │   ブラウザ       │   │
│  │ (ターミナル)  │  │  (Vercel URL)   │   │
│  │             │  │                 │   │
│  └─────────────┘  └─────────────────┘   │
│                                          │
│  ┌──────────────────────────────────┐    │
│  │   プロンプト入力エリア（拡大）      │    │
│  └──────────────────────────────────┘    │
│                                          │
└──────────────────────────────────────────┘
```

#### ソース追加手順（OBS）
1. **ソース1: Claude Code（左上）**
   - 「ウィンドウキャプチャ」を追加
   - ターミナル（Windows Terminal等）を選択
   - 位置: X=0, Y=0 / サイズ: 940x540 くらい

2. **ソース2: ブラウザ（右上）**
   - 「ウィンドウキャプチャ」を追加
   - ブラウザウィンドウを選択
   - 位置: X=960, Y=0 / サイズ: 960x540 くらい

3. **ソース3: プロンプト入力（中央下）**
   - 「ウィンドウキャプチャ」で同じターミナルを選択
   - クロップ（Alt+ドラッグ）でプロンプト入力部分だけ切り出す
   - 位置: X=160, Y=560 / サイズ: 1600x480 くらい
   - ※ターミナルの下部（入力行付近）だけが見えるように調整

> **Tip**: ソース3はソース1と同じウィンドウの別クロップ。
> プロンプト入力時に「何を打っているか」が大きく見える効果を狙う。

## 録画前チェックリスト

### 環境
- [ ] デスクトップ通知を全OFF（集中モード / おやすみモード）
- [ ] 不要なアプリを閉じる（Slack、Teams、メール等）
- [ ] 壁紙をシンプルなものに変更（映り込み防止）
- [ ] バッテリー残量確認 or 電源接続

### ターミナル
- [ ] フォントサイズ: 16px 以上（見やすさ重視）
- [ ] 背景色: 暗め（黒 or ダークテーマ）
- [ ] プロンプト表示がシンプル（長いパス表示はNG）
- [ ] 履歴クリア: `clear` しておく

### ブラウザ
- [ ] VercelデプロイURLを開いておく
- [ ] ブックマークバー非表示（Ctrl+Shift+B）
- [ ] 拡張機能アイコンを非表示に
- [ ] タブは1つだけ（余計なタブを閉じる）
- [ ] ズーム: 100%

### プロジェクト
- [ ] `git status` でクリーンな状態を確認
- [ ] `src/main.ts` がプレースホルダー状態であること
- [ ] `npm run dev` で開発サーバーが起動すること
- [ ] 3つのプロンプトをどこかにコピペ準備（メモ帳等）

## 録画の進行

### テイク前
```bash
cd C:\Users\shibu\dev\ai-game-demo
clear
```
→ OBS「録画開始」

### Phase 1: Prompt 1
1. ターミナルで `claude` を起動（既に起動済みならスキップ）
2. プロンプトを貼り付け（ゆっくりめに）
3. AIが動く様子を待つ（左画面）
4. 完了後 `git add . && git commit -m "feat: add game canvas and robot character" && git push`
5. ブラウザをリロード（右画面）
6. キーボードで操作デモ（5〜10秒）

### Phase 2: Prompt 2
1. 続けてPrompt 2を貼り付け
2. AIが拡張する様子を待つ
3. `git add . && git commit -m "feat: add platforms and gear items" && git push`
4. ブラウザリロード
5. ギアを集めてスコア増加を見せる（5〜10秒）

### Phase 3: Prompt 3
1. Prompt 3を貼り付け
2. AIが仕上げる様子を待つ
3. `git add . && git commit -m "feat: add obstacles, game over and title screen" && git push`
4. ブラウザリロード
5. タイトル → ゲーム開始 → プレイ → ゲームオーバー → リスタートを見せる

### 終了
→ OBS「録画停止」

## 動画編集のポイント

### カット推奨箇所
- `npm install` の待ち時間
- Vercelデプロイの待ち時間（30秒〜1分）
- AIの長い処理時間（早送りも可）
- タイプミスの修正部分

### 残すべき箇所
- プロンプト入力の瞬間（視聴者が読める速度で）
- AIがファイルを生成/編集する様子（数秒でOK）
- デプロイ完了 → ブラウザリロードの瞬間
- 実際のゲーム操作

### 仕上げ
- 必要に応じてBGM追加（フリー素材）
- テロップで「Prompt 1/3」「Prompt 2/3」「Prompt 3/3」を入れると分かりやすい
