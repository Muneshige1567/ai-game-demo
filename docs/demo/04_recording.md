# 04. 画面録画ガイド

## 録画ソフト

### OBS Studio（推奨・無料）
- https://obsproject.com/
- Windows/Mac 対応
- シーン切り替え、ウィンドウキャプチャが柔軟

### 代替
- Windows: Xbox Game Bar（Win + G）— シンプルだが画面構成が制限される
- Mac: QuickTime / ScreenFlow

---

## OBS 設定

### 1. 出力設定
- 設定 → 出力
  - 出力モード: 詳細
  - エンコーダ: x264 または NVENC（GPU搭載の場合）
  - ビットレート: 6000 Kbps（YouTube想定）
  - キーフレーム間隔: 2
- 設定 → 映像
  - 基本解像度: 1920x1080
  - 出力解像度: 1920x1080
  - FPS: 30

### 2. シーン構成

```
┌──────────────────────────────────────────┐
│                                          │
│  ┌─────────────┐  ┌─────────────────┐   │
│  │             │  │                 │   │
│  │ Claude Code │  │   ブラウザ       │   │
│  │ (ターミナル)  │  │  (Vercel URL)   │   │
│  │             │  │                 │   │
│  └─────────────┘  └─────────────────┘   │
│                                          │
└──────────────────────────────────────────┘
```

- 左: ターミナル（Claude Code実行画面）
- 右: ブラウザ（VercelデプロイURL）
- ※ 旧構成のプロンプト拡大エリアは不要（ワンショットなので入力は一瞬）

## 録画前チェックリスト

### 環境
- [ ] デスクトップ通知を全OFF（集中モード / おやすみモード）
- [ ] 不要なアプリを閉じる（Slack、Teams、メール等）
- [ ] 壁紙をシンプルなものに変更（映り込み防止）

- [ ] フォントサイズ: 16px 以上（見やすさ重視）
- [ ] 背景色: 暗め（黒 or ダークテーマ）
- [ ] プロンプト表示がシンプル（長いパス表示はNG）
- [ ] 履歴クリア: `clear` しておく

### ブラウザ
- [ ] VercelデプロイURLを開いておく
- [ ] ブックマークバー非表示（Ctrl+Shift+B）
- [ ] 拡張機能アイコンを非表示に
- [ ] タブは1つだけ（余計なタブを閉じる）
- [ ] ズーム: 100%

### プロジェクト
- [ ] `git status` でクリーンな状態を確認
- [ ] `ls` で GAME_SPEC.md のみ存在すること（他のファイルがないこと）
- [ ] ワンショットプロンプトをコピペ準備

## 録画の進行

### テイク前
```bash
cd C:\Users\shibu\dev\ai-game-demo
clear
```
→ OBS「録画開始」

### Phase 1: 空っぽを見せる
1. `ls` を実行 → GAME_SPEC.md しかないことを見せる
2. `cat GAME_SPEC.md` で仕様書をチラ見せ（2〜3秒）

### Phase 2: ワンショット実行
1. ターミナルで `claude` を起動
2. プロンプトを貼り付け:
   ```
   @GAME_SPEC.md この仕様書に従って、プロジェクトのセットアップからゲームの実装まで全て行ってください。
   タイトル画面、ゲームプレイ、クリア画面、ゲームオーバー画面まで全て実装してください。
   完成したらデプロイしてください。
   ```
3. AIがプロジェクトセットアップから始める様子を待つ（ここは倍速編集）
4. 完了後 → git push → Vercelデプロイ
5. ブラウザをリロード

### Phase 3: 完成デモ
1. タイトル画面を見せる
2. スペースでゲーム開始
3. 手動で少し操作（動くことを見せる程度でOK）

### Phase 4: AIプレイ（クライマックス）
1. AI自動プレイモードに切り替え
2. ロボットが自律的にギアを集め、障害物を避ける様子を見せる
3. 「AIが作って、AIが遊ぶ」の絵を撮る
4. 10秒程度で十分

### 終了
→ OBS「録画停止」

## 動画編集のポイント

### 目標: 約1分に収める

| セクション | 素材尺 | 編集後 | 処理 |
|:----------|:-------|:------|:-----|
| 導入テロップ | - | 5秒 | テロップ挿入 |
| 空っぽを見せる + 仕様書チラ見せ | 5秒 | 5秒 | 等速 |
| プロンプト入力 | 10秒 | 5秒 | 等速〜軽く倍速 |
| AI実装中 | 3〜5分 | 15秒 | 4〜8倍速 |
| デプロイ待ち | 30秒〜1分 | カット | カット |
| 手動操作 | 10秒 | 5秒 | 等速 |
| AIプレイ | 15秒 | 10秒 | 等速（クライマックス） |
| 締めテロップ | - | 5秒 | テロップ挿入 |

### 残すべき箇所
- プロンプト入力の瞬間（1行だけ！という驚き）
- コードが次々生成される様子（倍速でもインパクトあり）
- 完成したゲームの操作（等速で質を見せる）

### 仕上げ
- BGM追加（フリー素材、テンポ速めがマッチ）
- 冒頭と末尾にテロップ/CTA
- 「仕様書1枚 x ワンショット = 完成」のメッセージを強調
