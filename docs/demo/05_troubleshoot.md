# 05. トラブルシューティング & リカバリ

## 録画中に問題が起きた場合

### Claude Codeがエラーを出した
- Webinar広告用なのでリテイク推奨（エラー対応シーンは尺を食う）
- どうしても使う場合: 「修正してください」の一言で直る様子を倍速で

### ゲームが正しく動かない
| 症状 | 追加プロンプト |
|------|--------------|
| 画面が真っ白 | `Canvasが表示されません。デバッグして修正してください` |
| キャラが動かない | `矢印キーでロボットが動くようにしてください` |
| 足場をすり抜ける | `プラットフォームの衝突判定を修正してください` |
| ギアが取れない | `ギアの当たり判定が機能していません。修正してください` |
| ゲームオーバーにならない | `電撃ビームに触れたらゲームオーバーになるようにしてください` |
| タイトル画面がない | `タイトル画面を追加してください` |

### ワンショットで全機能が揃わなかった場合
- 不足分だけ短い追加プロンプトで対応
- 動画では追加指示部分をカット or 倍速で処理
- 最終成果物が完成していればOK

### 録画をやり直したい（ブランチリセット）
```bash
# デモブランチを破棄して新しく切り直す
git checkout master
git branch -D demo/run-YYYYMMDD
git checkout -b demo/run-YYYYMMDD-v2
# 再度 GAME_SPEC.md だけの状態にする（00_setup.md 参照）
```

## 事前リハーサルのすすめ

本番前に1回通しでやっておくと安心:

1. ワンショットプロンプトを実行
2. 所要時間を計測（倍速の速度決めに使う）
3. 不足機能があればプロンプトの調整
4. リハで出たバグの修正プロンプトをメモしておく

リハ後にリセットして本番に臨む。
