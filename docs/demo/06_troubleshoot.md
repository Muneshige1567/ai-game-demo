# 06. トラブルシューティング & リカバリ

## 録画中に問題が起きた場合

### Claude Codeがエラーを出した
- **そのまま活かす**: 「エラーが出ましたが、AIに修正を指示するだけで直ります」と見せる
- 修正プロンプト例: `エラーが出ています。修正してください`
- これも含めてデモのリアリティになる

### Vercelデプロイが失敗した
- `npm run build` をローカルで実行してビルドエラーを確認
- Claude Codeに `ビルドエラーを修正して` と指示
- 再度 push

### ゲームが正しく動かない
| 症状 | 追加プロンプト |
|------|--------------|
| 画面が真っ白 | `Canvasが表示されません。デバッグして修正してください` |
| キャラが動かない | `矢印キーでロボットが動くようにしてください` |
| 足場をすり抜ける | `プラットフォームの衝突判定を修正してください` |
| ギアが取れない | `ギアの当たり判定が機能していません。修正してください` |
| ゲームオーバーにならない | `電撃ビームに触れたらゲームオーバーになるようにしてください` |

### 録画をやり直したい（プロジェクトリセット）
```bash
# 初期状態に戻す
git log --oneline           # 初期コミットのハッシュを確認
git reset --hard <hash>     # 初期状態にリセット
git push --force            # リモートも戻す
```

> **注意**: force push は他のブランチに影響しないか確認してから実行

## 事前リハーサルのすすめ

本番前に1回通しでやっておくと安心:

1. 3つのプロンプトを通して実行
2. 各ステップの所要時間を計測
3. 問題が起きやすい箇所を特定
4. リハで出たバグの修正プロンプトをメモしておく

リハ後にリセットして本番に臨む。
