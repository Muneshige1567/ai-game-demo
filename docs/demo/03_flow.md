# 03. デモ全体フロー（タイムライン）

## コンセプト
- Webinar広告用デモ動画
- 目標尺: 約1分（倍速編集あり）
- 空のディレクトリ + 要件定義書1枚 → 完成ゲームのインパクト

## デモ全体の流れ

```
[導入] → [空っぽを見せる] → [ワンショット実行] → [完成デモ] → [AIプレイ] → [まとめ]
  5秒       5秒              25秒(倍速)          10秒        10秒       5秒
```

---

### Phase 0: 導入（〜5秒）
- テロップ: 「仕様書1枚からゲームを作る」的なキャッチ
- 画面構成の一瞬見せ（左: ターミナル / 右: ブラウザ）

### Phase 1: 空っぽを見せる（〜5秒）
- `ls` でディレクトリに GAME_SPEC.md しかないことを見せる
- GAME_SPEC.md を一瞬チラ見せ（仕様書の存在を示す）
- **見せ場**: 「本当にこれしかありません」感

### Phase 2: ワンショット実行（〜25秒 ※倍速）
- プロンプト入力（短い！）
- Claude Codeがプロジェクトセットアップから始める
  - package.json 生成、npm install、tsconfig、vite.config...
  - ゲームコード一式を生成
  - AI自動プレイモードも実装
- `npm run dev` → ブラウザで即確認
- **ここは4〜8倍速で編集**
- **見せ場**: 何もない状態から大量のファイルが次々に生成される様子

### Phase 3: 完成デモ（〜10秒）
- ブラウザリロード → タイトル画面！
- スペースでゲーム開始
- 手動で少し操作して動くことを見せる
- **見せ場**: ゼロから完成したゲームが動いている

### Phase 4: AIプレイ（〜10秒）★クライマックス
- AI自動プレイモードに切り替え
- ロボットが自律的にギアを集め、障害物を避ける様子
- **見せ場**: 「AIが作ったゲームを、AIが遊んでいる」
- 視聴者にとっての驚きのピーク

### Phase 5: まとめ（〜5秒）
- テロップ: 「仕様書1枚、プロンプト1回。AIが作り、AIが遊ぶ。」的な締め
- CTA（Webinar参加への誘導等）

---

## 注意事項
- 開発サーバー起動はほぼ一瞬なのでカット不要
- AI処理中も倍速で見せる（止まっている画面は退屈）
- エラーが出た場合は動画ではカット、またはリテイク
- ゲーム操作のシーンは等速で見せる（成果物の質を伝えるため）
