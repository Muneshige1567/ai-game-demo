# 01. ゲーム要件定義書（デモ用）

この要件定義書をプロジェクトルートに `GAME_SPEC.md` として配置し、
ワンショットプロンプトから参照させる。

---

## GAME_SPEC.md の内容

```markdown
# Robot Factory - ゲーム仕様書

## 概要
工場で働くロボットがプラットフォームを移動してギア（歯車）を集める2Dプラットフォーマーゲーム。

## 技術仕様
- HTML5 Canvas（800x600）
- Vite + TypeScript
- 外部ライブラリ不使用

## キャラクター：ロボット
- 四角いボディ、丸い目2つ、頭にアンテナ
- ピクセルアート風のシンプルなデザイン
- 操作: 矢印キーで左右移動、スペースキーでジャンプ

## ステージ
- 背景: 工場風の暗めの色合い（ダークグレー系）
- 地面: 金属的な質感
- プラットフォーム: ベルトコンベア風の足場を複数配置
- 配置は遊びやすいバランスで

## アイテム：ギア（歯車）
- 足場の上に配置
- ロボットが触れると取得
- 取得時にキラッとしたエフェクト
- 画面上部にスコア表示

## 障害物：電撃ビーム
- ステージ上に配置（水平 or 垂直）
- 点滅アニメーション付き
- 触れるとゲームオーバー

## ゲームフロー
1. タイトル画面（"Robot Factory" ロゴ + "Press SPACE to Start"）
2. ゲームプレイ（ギアを集める）
3. 全ギア取得 → ステージクリア画面
4. 障害物に触れる → ゲームオーバー画面（スコア表示 + リスタート）

## 物理
- 重力処理（自然な落下）
- プラットフォームとの衝突判定
- 画面端の制限
```

---

## 事前配置手順

デモ本番前にリポジトリを空にし、`GAME_SPEC.md` のみの状態にする:

```bash
# プロジェクトルートの全ファイルを削除し、GAME_SPEC.md だけにする
cp docs/demo/GAME_SPEC.md ./GAME_SPEC.md
# src/, index.html, package.json, node_modules/ 等を全て削除
git add -A && git commit -m "reset: empty project with spec only" && git push --force
```

最終状態:
```
ai-game-demo/
└── GAME_SPEC.md    ← これだけ
```

これにより、Claude Codeが `@GAME_SPEC.md` で参照でき、
プロジェクトセットアップからゲーム実装まで全てを一気に行う演出になる。
