# 02. ワンショットプロンプト

## 目的
要件定義書だけがある空のディレクトリから、1回のプロンプトでゲームを完成させる。
「仕様書1枚から、プロジェクトセットアップも含めて全て生成される」インパクトを見せる。

## プロンプト（コピペ用）

```
@GAME_SPEC.md この仕様書に従って、プロジェクトのセットアップからゲームの実装まで全て行ってください。
タイトル画面、ゲームプレイ、クリア画面、ゲームオーバー画面まで全て実装してください。
実装が完了したら、AIが自動でゲームをプレイするデモモードも追加してください。
ゲームが正しく動作することを確認してください。
```

## Claude Codeがやること（期待）
1. Vite + TypeScript のプロジェクトセットアップ（package.json, tsconfig, vite.config 等）
2. npm install
3. ゲーム全体の実装（Canvas, キャラクター, ステージ, アイテム, 障害物, 画面遷移）
4. AI自動プレイモードの実装（ロボットが自律的にギアを集め、障害物を避ける）
5. npm run dev で動作確認

## 完了後の期待状態
- タイトル画面が表示される（"Robot Factory" + "Press SPACE to Start"）
- スペースキーでゲーム開始
- ロボットが矢印キーで移動、スペースでジャンプ（手動プレイ）
- ベルトコンベア風の足場が複数配置
- ギアを集めるとスコア増加 + エフェクト
- 電撃ビーム障害物（点滅アニメーション付き）
- 障害物接触でゲームオーバー画面（スコア + リスタート）
- 全ギア取得でクリア画面
- **AI自動プレイモード**: ロボットが自律的にプレイする（デモの締めに使用）

## デモ時の操作ポイント
1. `ls` で GAME_SPEC.md しかないことを見せる（空っぽ感の演出）
2. プロンプトを入力（短い！）
3. Claude Codeがプロジェクトセットアップから始める様子を見せる
4. 大量のファイルが次々に生成される様子（倍速）
5. `npm run dev` → ブラウザで完成したゲームを確認
7. **AI自動プレイモードを起動 → AIが自分で作ったゲームを遊ぶ姿で締め**

## トラブル時の対応
- 一部機能が欠けていた場合 → 「タイトル画面が動いていないので修正して」等の短い追加指示
- ビルドエラー → 「ビルドエラーを修正して」で対応
- AI自動プレイが動かない → 「自動プレイモードでロボットが動くように修正して」
